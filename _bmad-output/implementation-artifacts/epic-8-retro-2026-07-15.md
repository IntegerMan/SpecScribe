## What Went Well

- **Seam discipline over spectacle.** One status model (8.2), one count ledger (8.3), paired progress/readiness (8.4), one primary next-step command (8.5), designed empty states (8.6), one primary requirements view (8.7), deterministic recency (8.8).
- **Story 8.1 spike paid off.** Surface map (shared path vs progressive vs CLI-notice) was injected into every sibling's Dev Notes. Prevented footer-only legend (8.2), set JS/webview expectations (8.4–8.6), and validated pure-CSS `:has()` for webview (8.7).
- **Visual-intent elicitation practiced successfully.** Owner silhouettes (hero primary + quiet alternates, dashed no-plan, radio toggle, absolute dates) largely survived review — unlike Epic 7 Story 7.1's post-review redesign rounds.
- **Split, don't absorb held hard.** 8.4 → 9.4 evidence strip; 8.5 → 8.6 empty states; 8.7 left Deep Analytics alone; 8.8 stayed on epic story cards + Commits tile; 8.3 deferred board card-set unification.
- **Truthfulness over convenience.** Defined ≠ Tracked; unrecognized ≠ drafted; absolute dates ≠ "Nd ago"; empty states keep the zero.

## Challenges

- **Progressive enhancement rediscovery.** Webview lacks `specscribe.js`; every JS tip/copy path needed a webview-safe channel (native `title`, always-visible legend, text-first). Known from 8.1, still re-decided story-by-story.
- **Silent wrong-path still appears in review.** Same family as Epic 3's "what does this silently drop?" — 8.8 nearly used top-N `Insights.Files` instead of uncapped `CodeMapMetrics`; 8.3 Defined/Tracked divergence; 8.2 unrecognized vs drafted.
- **Action items / retro commitments invisible in the sunburst.** Project Lead's primary "what's left" surface only knows epics/stories/tasks. Open process commitments live on follow-ups pages and home callouts — a traceability hole for debt follow-through.
- **Carried debt.** `KnownIndexGroups` omitting `adrs`/`retros` still open since Epic 4. Visual-intent create-story *process* fix still open (practice worked this epic; institutionalization did not).

## Key Insights

1. **A simple epic can still surface a product gap.** Epic 8 made story status trustworthy; it did not make open retro commitments visible in the Driver's remaining-work geometry.
2. **Traceability includes process follow-through.** Action items born in retros belong in Epic 9's charter alongside requirement → story hops — not as side-quest backlog.
3. **Spike + visual-intent-up-front + split-don't-absorb is a winning trio** for seam epics. Formalize visual-intent in create-story so Epic 8's success is not accidental.
